# P4 Roadmap — Prompt & Behavior Tuning

> 목표: 에이전트 응답 품질/일관성 개선 (과잉 적극성 제어 + 프롬프트 표준화)

---

## Phase 1: 기본 프롬프트 템플릿 시스템

**목표:** 새 에이전트 생성 시 AGENTS.md 품질을 기본 보장

- [ ] 기본 AGENTS.md 템플릿 내장 (역할/톤/경계/보고 형식/실패 시 행동)
- [ ] 템플릿 프리셋 제공: `conservative` / `balanced` / `proactive`
- [ ] 생성 시 `agentsMd`가 비어있거나 너무 짧으면 템플릿 자동 적용
- [ ] 기존 에이전트 호환 유지 (명시 입력 우선)

## Phase 2: 메모리 부트스트랩 표준화

**목표:** 생성 직후부터 메모리 루틴 동작 가능

- [ ] 생성 시 `MEMORY.md` 자동 생성
- [ ] 생성 시 `memory/` 폴더 + `memory/YYYY-MM-DD.md` 자동 생성
- [ ] 세션 시작 시 today/yesterday/MEMORY 로드 규칙 점검 및 보강
- [ ] 메모리 파일이 없어도 안전하게 동작 (graceful fallback)

## Phase 3: Skills 확장성 기반

**목표:** 사용자별 커스텀 스킬 확장 경로 제공

- [ ] 생성 시 `skills/` 폴더 자동 생성
- [ ] `skills/README.md` 자동 생성 (구조/작성법/예시)
- [ ] 커스텀 스킬 검색 경로 및 로딩 동작 문서화

## Phase 4: 응답 적극성 제어

**목표:** 필요할 때만 말하고, 말할 땐 정확히 말하도록

- [ ] "응답 여부 판단" 가이드라인 강화 (질문/멘션/긴급도/신규 정보)
- [ ] DM vs 그룹 정책 분리 템플릿 제공
- [ ] 과잉 응답 방지 규칙 추가 (중복 응답/잡담 과개입 방지)
- [ ] 응답 길이/빈도 프리셋 반영

## Phase 5: 평가 & 회귀 테스트

**목표:** 품질을 수치화하고 회귀 방지

- [ ] 시뮬레이션 대화셋 구축
- [ ] 지표: 과잉응답률 / 누락률 / 보고 포맷 일관성
- [ ] 각 Phase 종료 시 빌드 + 테스트 + 리뷰 + 체크리스트 완료

---

## 운영 원칙

- 각 Phase는 다음 순서로 진행:
  1. 구현
  2. Codex 리뷰(수정 포함)
  3. 빌드/테스트 검증
  4. 커밋 + 요약 보고
- 위험 변경(삭제/마이그레이션)은 사전 보고 후 진행
