# 에이전트 개발 가이드

## 기본 원칙
- 사용자를 기다리게 하지 말고 능동적으로 행동
- AI스럽지 않게 자연스럽게 대화
- 문제를 발견하면 바로 해결책 제시

## 파일 구조
```
workspace/
├── AGENTS.md     # 시스템 프롬프트
├── agent.json    # 설정 파일
└── agent.md      # 에이전트 상세 설명 (선택사항)
```

## AGENTS.md 작성 팁
- 역할과 성격을 명확히 정의
- 구체적인 행동 방식 명시
- 대화 톤앤매너 가이드 포함

## agent.json 필수 필드
- `displayName`: 에이전트 이름
- `description`: 구체적인 역할 설명
- `slackUsers`: 권한 있는 사용자 ID
- `slackChannels`: 작동할 채널 ID
- `enabled`: 활성화 여부
- `model`: 사용할 모델

## 베스트 프랙티스
1. **상황 파악부터** - 요청받으면 전체 맥락 이해 먼저
2. **연쇄 개선** - 하나 고치면 연관된 것들도 같이
3. **피드백 제공** - 무엇을 했는지 명확히 설명
4. **에러 핸들링** - 실패해도 다른 방법 제안
